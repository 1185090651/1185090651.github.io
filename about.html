                          # Sequelize

                          # 为什么要使用Sequelize？

                          • 平常系统中的增删改查、设置外键、连表查询等操作都比较繁琐，需要记忆SQL语句
                          • 需要手动执行和创建数据表
                          • 前端调用api直接操作数据库，缺乏安全性

                          # ORM - Object Relational Mapping

                          • 通过对象关系的形式将数据映射到数据库
                          • 通过操作对象来操作数据库
                          • 可以规范数据结构
                          • 数据表，用JS中的模型（class或对象）代替
                          • 一条或多条记录，用JS中的一个对象或数组代替
                          • SQL语句，用对象方法代替

                          # 环境准备

                          安装sequelize及依赖的mysql2

                          npm i sequelize mysql2 -S

                          # 建模（外键）& 同步到数据库

                          # 1. 创建连接实例，并模块化

                          新建seq.js文件创建连接实例，后面我们会多次使用这个实例进行创建模型、同步数据等等，所以将此单独写成一个模块

                          const Sequelize = require('sequelize')

//实例配置
const conf = {
    host: 'localhost', // 主机名
    dialect: 'mysql' // 操作的数据库类型
}

// 创建实例 
const seq = new Sequelize('seq_test', 'root', 'mysqlroot', conf) // 数据库名 用户名 密码 配置

module.exports = seq;

                          我们可以使用Sequelize中的如下方法测试连接是否成功

                          // 测试连接
seq.authenticate().then(() => {
    console.log('ok');
})
.catch(() => {
    console.log('err');
})

                          # 2. 创建所需的数据模型

                          新建model.js将抽象的数据模型用代码体现出来

                          const Sequelize = require('sequelize')
const seq = require('./seq')

// 创建 User 模型  数据表名users(与mongoose相似)
const User = seq.define('user', { 
    // id 会自动创建，并设为主键自增
    userName: {
        type: Sequelize.STRING, // varchar(255) 255个长度字符
        allowNull: false // 是否可为空
    },
    password: {
        type: Sequelize.STRING,
        allowNull: false
    },
    nickName: {
        type: Sequelize.STRING
    }
    // 自动创建 createdAt 和 updatedAt
})

const Blog = seq.define('blog', {
    title: {
        type: Sequelize.STRING,
        allowNull: false
    },
    content: {
        type: Sequelize.TEXT,
        allowNull: false
    },
    userId: {
        type: Sequelize.INTEGER,
        allowNull: false
    }
})

module.exports = {
    User,
    Blog
}

                          我们也可以在代码中定义外键

                          // 创建外键关联
Blog.belongsTo(User, { // 多对一
    // 创建外键 Blog.userId => User.id
    foreignKey: 'userId',
    onDelete: 'cascade'
})

User.hasMany(Blog, { // 一对多
    // 创建外键 Blog.userId => User.id
    foreignKey: 'userId'
})

                          虽然只是一个博客id与用户id的关联，但是我们需要为这两个模型都创建关联，否则在后面的连表查询会找不到两个表的关系。onDelete属性是外键约束的类型，cascade是指从父表中删除或更新对应的行，同时自动的删除或更新自表中匹配的行。

                          # 3. 同步数据表结构

                          新建sync.js,将刚才定义好的模型同步到数据库中

                          const seq = require('./seq')

require('./model') // 导入模型

// 测试连接
seq.authenticate().then(() => {
    console.log("auth ok");
})
.catch((err) => {
    console.log(err);
})

// 同步
seq.sync({ force: true }).then(() => { // 强制同步
    console.log("sync ok");
    process.exit()
})

                          # 连接池

                          在以上开发中，每一个用户进来都会和数据库创建一个连接，连接数多了会消耗很大的性能。所以在线上我们采用连接池的方式来做。连接池存在于内存中，他可以根据用户的连接数动态创建连接，等待用户的使用。每个用户进来都会使用连接池的连接与数据库关联，当连接数超过连接池的最大连接数的时候，下一个用户的连接会等待排队，当没有用户连接时，连接池中的连接会自动释放。

                          连接池的配置比较简单，就是在连接配置中加入pool属性，当项目处在线上添加上此配置即可

                          // 线上环境 使用连接池
conf.pool = {
    max: 5, // 连接池中最大的连接数量 (根据实际用户访问量动态修改)
    min: 0, // 最小
    idle: 10000 // 如果一个连接池 10 s 之内没有被使用，则释放
}
