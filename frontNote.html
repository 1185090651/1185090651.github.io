# 前端面试

# HTML

# 1. 请讲一下HTML5的特点

  • HTML5的跨平台技术 HTML5技术跨平台，适配多终端。传统移动终端上的Native App，开发者的研发工作必须针对不同的操作系统进行，成本相对较高。Native App对于用户还存在着管理成本、存储成本以及性能消耗成本。HTML/JavaScript/CSS语言所开发的应用只要一次开发就能进入所有浏览器进行分发。即使是走传统的App Store应用商店渠道，只需要再将底层用HTML5开发的应用“封装“为App，从时间和资金成本上讲远小于跨系统移植。
  • HTML5更加的用户体验 提高了用户体验，加强了视觉感受。HTML5在移动端能够让应用程序回归网页，并对网页的功能进行扩展，用户不需要下载客户端或者插件就能观看视频、玩游戏，操作更加简单，用户体验更好。HTML5的视音频新技术解决了移动端苹果和安卓4.0+对Flash的不支持问题，HTML5在性能上要比Flash更好，让用户拥有更好的体验。HTML5中的CSS3特效样式、Canvas、WEBGL的介入，不仅加强了网页的视觉效果，甚至能够使用户在网页当中看到三维立体特效。
  • HTML5的语言与推动人群——发展比较平稳 HTML5是由多种语言结合而成的，每种语言在功能发挥中各行其责。多种的语言也就决定着知识的广度是比较大的，同时可以根据自己的兴趣方向以及自己对知识的掌握水平进行选择，找到适合自己的方向的职位就业。 HTML5的始创来源于谷歌的苹果为首成立的WHAT WG组织，进行HTML5技术的研究。2009年，W3C将HTML5纳入到推进标准当中。随后，国内百度、腾讯等各大互联网巨头蜂拥与W3C组织合作，宣布参与HTML5标准定制。同时，W3C组织也努力让开发者、用户参与其标准制定当中。也就是说，来自浏览器、开发者、用户等各方面的力量共同推动HTML5向前发展。 --转载自HTML5学堂（码匠）

# 2. 请简单介绍下行内元素和块级元素

  • 定义

    **行内元素：**一个行内元素只占据它对应标签的边框所包含的空间

    **块级元素：**块级元素占据其父元素（容器）的整个空间，因此创建一个“块”。通常浏览器会在块级元素前后另起一个新行。

  • 特点

    行内元素：

    • 和其他元素都在同一行上
    • 高、行高、外边距和内边距不可改变
    • 宽度就是她的文字或者图片的宽度，不可改变
    • 行内元素只能容纳文本或者其他行内元素
    • 设置行内元素，需要注意如下：
      • 设置宽度 width 无效
      • 设置高度 height 无效，但可以通过 line-height 来设置
      • 设置 margin 只有左右有效，上下无效
      • 设置 padding 只有左右有效，上下无效。注意元素范围是增大了，但是对元素周围的内容是没影响的。

    块级元素：

    • 总是在新一行开始
    • 高度、行高、外边距和内边距都可改变
    • 若宽度没有设置，则默认宽度为容器的100%，除非设置一个宽度
    • 块级元素可以容纳行内元素和其他块级元素

  • 行内元素和块级元素的转换

    • display

      (1) 行内元素 display:inline;

      (2) 块级元素 display:block;

      (3) 行内块元素 display:inline-block;

      (4) 行内元素转换块级元素 display:block;

      (5) 块级元素转换行内元素 display:inline;

    • float

      若设置行内元素 float:left/right;则该行内元素转换为块级元素，且具有浮动特性。

    • position

      若为行内元素进行定位，position:absolute;或者position:fixed都会把行内元素转换为块级元素。

# 4. XHTML和HTML的区别

  • **XHTML：**可扩展超文本标记语言，XHTML元素必须关闭。标签名必须用小写字母。XHTML文档中元素必须被嵌套于根元素中。
  • **HTML：**超文本标记语言，在HTML中允许一些不规范的写法，HTML对于各大浏览器兼容性较差，现在web前端开发的静态网页，一般都是HTML4，HTML5就另当别论了。

# CSS

# 1. position、relative和absolute分别是相对于谁进行定位的？

  • absolute：生成 绝对定位 的元素，相对于最近一级的定位不是static的父元素来进行定位。
  • fixed（老IE不支持）：生成 绝对定位 的元素，通常相对于浏览器窗口或frame进行定位。
  • relative：生成 相对定位 的元素，相对于其在普通流的位置进行定位，跟父元素没有关系。不随滚动条的滚动而滚动，固定在浏览器可是窗口的位置。
  • static：默认值 没有定位 ，元素出现在正常的文档流中。相当于none没有定位的意思。（没有偏移我们几乎不用）
  • sticky：生成 粘性定位 的元素，容器的位置根据正常文档流计算得出（基于用户的滚动位置来定位）

    粘性定位的元素是依赖于用户的滚动，在position:relative;于position:fixed;定位之间的切换。它的行为就像position:relative;而当页面滚动超出目标区域时，它的表现就像position:fixed;它会固定在目标位置。它有如下特性： 1.sticky不会触发BFC 2.样式表z－index无效。行内style写有效 3.sticky是容器相关的，也就说sticky的特性只会在他所处的容器里生效。（即使元素已经到了达到了fixed的要求,但是不会跑到父元素之外,会优先被父元素的边界带着跑）

# CSS隐藏元素 display、visibility、opacity的区别

# display: none

  • DOM 结构：浏览器不会渲染 display 属性为 none 的元素，不占据空间；
  • 事件监听：无法进行 DOM 事件监听；
  • 性能：动态改变此属性时会引起重排，性能较差；
  • 继承：不会被子元素继承，毕竟子类也不会被渲染；
  • transition：transition 不支持 display。

# visibility: hidden

  • DOM 结构：元素被隐藏，但是会被渲染不会消失，占据空间；
  • 事件监听：无法进行 DOM 事件监听；
  • 性 能：动态改变此属性时会引起重绘，性能较高；
  • 继 承：会被子元素继承，子元素可以通过设置 visibility: visible; 来取消隐藏；
  • transition：visibility 会立即显示，隐藏时会延时。

# opacity: 0

  • DOM 结构：透明度为 100%，元素隐藏，占据空间；
  • 事件监听：可以进行 DOM 事件监听；
  • 性 能：提升为合成层，不会触发重绘，性能较高；
  • 继 承：会被子元素继承,且，子元素并不能通过 opacity: 1 来取消隐藏；
  • transition：opacity 可以延时显示和隐藏
